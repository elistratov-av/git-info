# Что вы узнали

Ещё одна тема о Git позади. Подведём итоги и повторим основные команды, которые встретились вам в уроках.

## Инициализация репозитория
```git init``` (от англ. _initialize_, «инициализировать») — инициализируй репозиторий.

## Подготовка файла к коммиту

```git add todo.txt``` (от англ. _add_, «добавить») — подготовь файл ```todo.txt``` к коммиту;

```git add --all``` (от англ. _add_, «добавить» + all, «всё») — подготовь к коммиту сразу все файлы, в которых были изменения, и все новые файлы;

```git add .``` — подготовь к коммиту текущую папку и все файлы в ней.

## Создание коммита

```git commit -m "Комментарий к коммиту."``` (от англ. _commit_, «совершать», «фиксировать» + message, «сообщение») — сделай коммит и оставь комментарий, чтобы было проще понять, какие изменения внесены. 

## Просмотр информации о коммитах

```git log``` (от англ. _log_, «журнал [записей]») — выведи подробную историю коммитов.

## Просмотр состояния файлов

```git status``` (от англ. _status_, «статус», «состояние») — покажи текущее состояние репозитория.

С этим набором команд вы сможете самостоятельно инициализировать проект и начать отслеживать в нём изменения. Фундамент знаний о Git заложен!
